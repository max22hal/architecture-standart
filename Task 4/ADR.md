### <a name="_qmphm5d6rvi3"></a>**Решение**
Диаграмма контекста:

Диаграмма контейнеров:

Для расширения функционала нужно доработать систему кол-центра. Она должна быть в состоянии принимать из Kafka и хранить информацию о ставках за сегодняшний день. Банк обладает компетенцией для доработки системы, поэтому ее можно реализовать внутренними силами.
Для партнерской системы нужно разработать отдельный микросервис, который будет считывать информацию о ставках из Kafka, формировать CVC файл и выгружать их в Yandex Cloud. После чего сервис отправляет E-mail по протоколу SMTP в партнерский кол-центр. В письме находится ссылка на хранилище Yandex cloud. Это решение позволят быстро и дешево реализовать желаемый функционал и, в будущем, безболезненно вывести его из эксплуатации.
АБС нужно доработать, чтобы она могла выгружать информацию по ставкам в Kafka каждый день.

### <a name="_bjrr7veeh80c"></a>**Альтернативы**
В качестве альтернативы можно использовать Python для реализации нового микросервиса. Это позволит ускорить разаботку, но в банке нет людей, обладающих нужной компетенцией, так что придется привлекать сторонних подрядчиков.
Другой вариант - можно сделать Serverless-функцию (Яндекс Cloud Functions), но в таком случае нужны люди с опытом работы с Yandex Cloud Functions.
Желаемый функционал можно реализовать процессуально. Менеджеры кол-центра могут вручную отправлять файлы со ставками в партнерский кол-центр, но это черевато ошибками.

**Недостатки, ограничения, риски**

Из недостатков можно отметить сложность реализации нового микросервиса. Это потребует дополнительного времени на разработку. Также, практически полностью отсутствует автоматизация и альтернативные каналы связи.
