| Код | Требования                         | Комментарий  |
|-----|------------------------------------|--------------|
| F   | Функциональные (Functionality)     |              |
| F1  | Клиент подает заявку на депозит, оставив свои контактные данные |              |
| F2  | Клиент может посмотреть список доступных депозитов с актуальными и персонализированными ставками |              |
| F3  | Указав счёт и сумму депозита, клиент может подать заявку на открытие депозита |  |
| F4  | Клиент получает СМС с кодом для подтверждения операции открытия депозита |  |
| F5  | Заявку на открытие депозита обрабатывает менеджер в бэк-офисе в АБС |  |
| F6 |  Клиент получает СМС с подтверждением открытия депозита |  |
|     | ...                                |              |
| U   | Удобство использования (Usability) |              |
| U1  | Интерфейс должен придерживаться системы дизайна для приложений, которая принята в компании. Пользователи при работе должны видеть привычные цвета и брендированные элементы |              |
|     | ...                                |              |
| R   | Надёжность (Reliability)           |              |
| R1  | Все сервисы должны работать 24/7 и быть доступны в 99,9% случаев |              |
| R2  | Не должно быть прямой работы интернет-банка с API АБС |              |
| R3  | В случае сбоев в ЦОД необходимо, чтобы сервисы интернет-банка были доступны и выдерживали требуемую нагрузку - необходимо обеспечить горячее резервирование на базе резервного ЦОДа | Для быстрого и бесшовного переключния на резервные мощности |
| P   | Производительность (Performance)   |              |
| P1  | Отклик системы по всем операциям должен занимать миллисекунды |              |
| P2  | Обеспечить горизонтальное масштабирование серверов, приложений и ЦОД  |              |
| S   | Поддерживаемость (Supportability)  |              |
| S1  | Должна быть поддержка интеграционного тестирования взаимодействия приложений  | Необходимо протестировать, что все элементы системы корректно взаимодействуют друг с другом |
| S2  | Сопровождение системы колл-центра | Возможно, система колл-центра потребует доработок. Необходимо обеспечить своевременную доработку и поддержку подрядчиком  |
| +R  | + Ограничения (Restrictions)       |              |
|  R1 | Данные клиента должны передаваться с сайта с помощью мезанизма шифрования трафика |              |
|  R2 | Ставки по депозитам хранятся только в XLS файлах                                |              |
|  R3 | Технологии должны быть ограничены уже имеющимися в банке. Например, БД MS SQL, Oracle, .Net Framework 4.5, React.js, Java Spring Boot, PHP |  |
|  R4 | Новые технологии должны быть совместимы с уже существующими |  |
|  R5 | Для ассинхронной коммуникации нужно использовать Kafka | Задел на перспективу - понадобится при переходе на микросервисную арзитектуру |
|  R6 | Текущая версия платформы интернет-банка не совместима с Kafka |  |
|  R7 | АБС может масштабироваться только вертикально из-за своей базы данных | Стоит рассмотреть возможность миграции АБС на горизонтально масштаируемую БД |
