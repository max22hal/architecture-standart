При выборе решения предпочтение отдавалось технологическому стеку, уже существующему в организации.
Для ускорения работы системы используется кеширование при помощи Redis.
В качестве очереди сообщений используется Kafka, как предпочтительное решение для дальнейшего развития системы. Так как АБС банка не совместима с Kafka напрямую, предлагается реализовать адаптер с использованием Debezium. Так как БД АБС не предполагает горизонтального масштабирования предлагается заложить дополнительный бюджет на вертикальное масштабирование и последующую миграцию АБС на другую БД.
Также для работы платформы интернет банка с Kafka нужно реализовать адаптер. Для этого можно использовать стандартную библиотеку Kafka Confluent.Kafka, так как она позволяет интегрировать Kafka в .Net приложения.
Вся коммуникация между сервисами реализованна через Kafka, таким образом достигается отказоустойчивость сервисов, масштабируемость и ассинхронность.
Для повышения доступности системы предлагается использовать резервный ЦОД. Резервный ЦОД должен быть в состоянии горячего резерва. Резервирование осуществляется при помощи Stretch Cluster - Kafka синхронно реплицирует данные на запасной ЦОД.
Взаимодействие клиента с интернет банком и сайтом банка происходит по протоколу HTTPS. Чувствительные данные шифуются при передаче между сервисами.

### <a name="_bjrr7veeh80c"></a>**Альтернативы**
В качестве альтернативы можно использовать Rabbit MQ. Это позволит избежать построения адаптеров для работы. Также для горячего резервирования можно использовать другие подходы, в частности MirrorMaker или Cluster Linking. Но такой подход может вести к некоторому отстованию резервного кластера

**Недостатки, ограничения, риски**

Из недостатков можно отметить сложность реализации горячего резерва. Это требует опыта работы с Kafka, которого в данный момент у компании нет. К тому же выбранный подход резервирования может вызывать отставание ЦОДов, в случае если они находятся на большом расстоянии друг от друга
Также монолитная АБС является слабым местом и ограничением. БД АБС не позволяет горизонтально масштабироваться. В качестве решения, стоит рассмотреть миграцию на другую БД и рефакторинг АБС с целью перехода на микросервисную архитектуру.
